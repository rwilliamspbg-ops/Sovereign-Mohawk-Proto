# Formal Go-Live Gate Artifacts

This directory stores machine-readable go-live evidence and attestation state.

## Primary Report

- `go-live-gate-report.json`
  - Generated by `python3 scripts/validate_go_live_gates.py`
  - Overall pass/fail plus detailed checks, warnings, and failures
  - Includes `host_preflight_mode` (`strict` or `advisory`)

## Attestations

Attestation files are in `attestations/` and are required by the go-live validator.

Each file includes:

- `gate`: gate identifier
- `status`: `pending` or `approved`
- `owner`: responsible owner/team
- `date`: approval date
- `evidence`: list of repository artifact paths
- `notes`: rationale/context

## Validation Command

Run from repository root:

```bash
make go-live-gate
```

Mode-specific commands:

```bash
make go-live-gate-strict
make go-live-gate-advisory
```

or directly:

```bash
python3 scripts/validate_go_live_gates.py
```

## Golden Path Evidence

- `golden-path-report.json`
  - Generated by `bash scripts/golden_path_e2e.sh`
  - End-to-end readiness, aggregation/lease checks, and metric assertion status
- `golden-path-report.md`
  - Human-readable summary of the same execution

## SLO Baseline and Failure-Injection Evidence

- `evidence/slo_sli_baseline_2026-03-28.md`
  - Versioned SLO/SLI contract for readiness, recovery latency, and proof latency
- `evidence/slo_sli_baseline_2026-03-28.json`
  - Machine-readable SLO threshold and policy source used by validators
- `evidence/failure_injection_latency_validation_2026-03-28.json`
  - Generated by `python3 scripts/validate_failure_injection_latency.py`
  - Includes per-scenario threshold checks against `chaos-reports/*-summary.json`
- `evidence/failure_injection_latency_validation_2026-03-28.md`
  - Human-readable summary table of failure-injection latency validation outcomes
